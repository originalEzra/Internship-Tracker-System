# 12.2 Invalid Status Transition Should Return 400

This request verifies that the backend owns internship status transition rules.
After step 12, the internship status is `ONLINE_ASSESSMENT`, so moving it back
to `DRAFT` should be rejected.

```http
PUT {{baseUrl}}/api/internships/{{internshipId}}
Authorization: Bearer {{token}}
Content-Type: application/json
```

Body:

```json
{
  "company": "Apifox Company Invalid Transition",
  "position": "Backend Intern Invalid",
  "location": "Sydney",
  "status": "DRAFT",
  "applicationUrl": "https://example.com/job-invalid"
}
```

Assertions:

```javascript
pm.test("invalid status transition is rejected", function () {
  pm.response.to.have.status(400);
  const json = pm.response.json();
  pm.expect(json.code).to.eql(400);
  pm.expect(json.message).to.include("Cannot change internship status from ONLINE_ASSESSMENT to DRAFT");
});
```
